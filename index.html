01

The Problem

Why do we need energy-efficient Graph Neural Networks?

Energy Crisis in AI

As graphs scale in size and GNN architectures grow in complexity, the energy budget required for training and inference becomes unsustainable — limiting deployment on edge devices and real-time systems.

Scale Challenge

Computational mechanics datasets often involve large irregular domains, molecular structures, and complex microstructures with hundreds of interacting elements — demanding efficient processing.

Regression Accuracy

Traditional spiking neurons with binary outputs struggle on regression tasks. The key challenge: how to achieve energy efficiency through sparsity without sacrificing prediction accuracy?

💡
Key Insight: The brain processes information using sparse electrical spikes — what if we could bring this biological efficiency to Graph Neural Networks while maintaining their regression accuracy?
02

Graph Neural Networks

Understanding the foundation: message passing on graphs

Graph Structure

A graph G = (V, E) consists of nodes V and edges E. In computational mechanics, nodes represent physical entities (grains, atoms) and edges represent connections (boundaries, bonds).

Node features: hi(l) ∈ ℝd

Message Passing

Each node collects "messages" from its neighbors. The message function computes information to send based on both the sender and receiver features.

Messages: mi(l) = Aj∈N(i) U₁(hi(l-1), hj(l-1), F̄ij)

Aggregation

Messages from all neighbors are combined using a permutation-invariant function (sum, mean, max, or combinations like PNA).

PNAConv: Uses mean, sum, min, max aggregators with degree-dependent scalers

Update

Each node updates its feature vector based on the aggregated messages and its own previous features.

Update: hi(l) = U₂(hi(l-1), mi(l))

GNN Variants Used in This Paper

SAGEConv

hi(l) = W₁hi(l-1) + W₂ · AGG(hj(l-1))

Samples and aggregates neighbor features with learnable weights. Used in Examples 1 & 2.

PNAConv

Multiple aggregators × degree scalers

Principal Neighborhood Aggregation — combines mean, sum, min, max with scaling. Used in Example 3.

Global Mean Pool

õ = (1/n) Σ hi(L)

Aggregates all node features to a single graph-level vector for scalar prediction tasks.

Spiking Neurons

From biological inspiration to Variable Spiking Neurons

LIF Neuron

Binary Spikes
1
Integrate: M(t̄) = βM(t̄-1) + z(t̄)
2
Fire: y(t̄) = 1 if M(t̄) ≥ Th, else 0
3
Reset: M(t̄) ← 0 after spike
⚠ Limitation: Binary outputs (0 or 1) lose magnitude information — poor for regression tasks.
VS

VSN (Variable Spiking Neuron)

Graded Spikes ✓
1
Integrate: M(t̄) = βM(t̄-1) + z(t̄)
2
Gate: ỹ(t̄) = 1 if M(t̄) ≥ Th, else 0
3
Graded Output: y(t̄) = σ(z(t̄) · ỹ(t̄))
✓ Key Innovation: Applies continuous activation σ to the gated input — preserving magnitude while maintaining sparse firing.

Surrogate Gradient for Training

The spike generation step is non-differentiable (a step function). During backpropagation, we approximate its gradient using a smooth fast-sigmoid surrogate function — enabling end-to-end training with standard optimizers.

04

The HVS-GNN Architecture

Selectively replacing activations for the optimal energy-accuracy tradeoff

The Hybrid Approach

Using only VSNs throughout degrades accuracy. Using no spiking neurons wastes energy. The HVS-GNN strategically places VSNs at select activation positions — achieving the best of both worlds.

Configuration Variants

Different positions of VSN replacement yield different accuracy-efficiency tradeoffs:

A-GNN
Baseline
All artificial activations
HLIF-GNN
LIF Variant
Binary spikes — limited regression ability
HVS-GNN
Proposed
Graded spikes — near-baseline accuracy + sparse

Spiking Loss Function (SLF)

To further promote sparsity, an optional penalty on spiking activity is added to the loss:

Lslf = αL · Lvanilla + βL · S
S = spikes produced / total possible spikes
αL, βL = tunable weights balancing accuracy vs. sparsity

Application Examples

Three challenging computational mechanics problems

α-Titanium Polycrystal Properties

Dataset 1,200 Microstructure Volume Elements
Textures 12 crystallographic (A–L)
Nodes Individual grains
Edges Shared grain boundaries
Node Features 4 orientation components + grain size (5D)
Outputs Stiffness (k) & Yield strength (fy)
GNN Type SAGEConv
Training 630 MVEs, LR=0.001, 600 epochs
L(32) → A₁ → SC(64) → A₂ → SC(64) → A₃ → GMP → L(64) → A₄ → L(16) → A₅ → L(1)

Terfenol-D Magnetostriction Prediction

Dataset 2,287 3D polycrystalline structures
Grains per structure 12–297 grains
Nodes Grains
Edges Grain boundaries
Node Features 3 Euler angles + size + neighbors (5D)
External Input Magnetic field direction
Output Effective magnetostriction λeffxx
Training 1,831 samples, LR=0.0005, 500 epochs
MP₁ → L(40) → A₁ → MP₂ → L(3) → A₂ → Flat+Cat → L(1024) → A₃ → L(128) → A₄ → L(1)

Graphene Membrane Stress Field

Dataset 2,000 porous graphene membranes
Membrane Size 127.9Å × 127.8Å
Nodes Individual atoms
Edges Atomic bonds (cutoff 1.92Å)
Input 2D atomic coordinates
Output Per-atom tensile stress field
GNN Type PNAConv + GRU (14 layers)
Training 1,400 samples, LR=0.001, 240 epochs
[ PNA(50) → GRU(50) → BN → A ] ×14 → PNA(1)

07

Key Takeaways

What makes HVS-GNN significant?

1

Graded Spikes Solve Regression

By applying a continuous activation to gated inputs, VSNs produce non-binary spikes that preserve magnitude information — enabling accurate regression unlike binary LIF neurons.

2

Hybrid Design is Optimal

Selectively placing VSNs (not everywhere) achieves the best accuracy-efficiency tradeoff. Full spiking hurts accuracy; no spiking wastes energy.

3

40–82% Sparsity

Across all three examples, HVS-GNN suppresses 40–82% of neuron activations, directly translating to energy savings — especially impactful on neuromorphic hardware.

4

Plug-and-Play Flexibility

Any existing GNN can be converted to its HVS variant with minimal effort — simply replace select activations with VSNs. No architectural redesign needed.
